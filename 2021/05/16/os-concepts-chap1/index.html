os_concepts_chap1

  • 数据从主存复制到寄存器和cache是由硬件控制的(没有OS干预), 主存和磁盘之间的数据对换是由OS控制的.
  • 多处理器环境下的cache一致性是在硬件层面保证的
  • 对称多处理(SMP): 每个CPU运行同一个OS备份, 每个处理器都是平等的, 都可以控制I/O, CPU之间通过系统总线进行通信
  • 非对称多处理: 有一个master和多个slave, master处理器分配任务给slave, I/O只能由master控制
  • network computer: 依靠一个中心计算机提供服务, 自己只保留一个最小化的OS
  • Interrupt 与 trap
    • Interrupt: 硬件中断, 时钟等
    • trap(exception): 软件中断, 系统调用, 算术错误等
  • DMA内存访问:
    • 高速的I/O设备直接访问内存, 不经过CPU
    • CPU写入某个特定的寄存器, 然后设备独立进行I/O任务, 完成后通过中断告知CPU.
    • DMA controller在写入数据到主存时, CPU正在运行的进程对于主存的访问可能会收到干扰, 因为两者需要争夺memory bus.
  • cpu记录每个进程可合法访问的内存区间: 用base和limits两个寄存器, 并在每次访问内存时进行check
  • cluster中的各台机器有各自的本地内存, 应用数据被存储在共享数据存储空间.

  • 网络应用之间在应用层进行通信
  • 网络应用层架构:
    • Client-server
      • Has an always-on host (server)
      • data center: housing a large number of hosts, creating a powerful virtual server.
      • Examples: Google, Youtube, Gmail, Facebook
    • P2P
      • 没有指定的server端, host之间之间进行通信
      • Traffic-intensive应用多采用P2P架构
      • Examples: file sharing(e.g. BitTorrent), Xunlei, Skype, IPTV.
  • 一些应用混合使用Client-server和P2P两种架构, 例如聊天软件, server端记录用户的IP地址, 用户之间的信息直接通过P2P发送.
  • Upstream traffic and downstream traffic的区别:

  • Socket: 应用层和传输层之间的接口

  • 应用层开发人员只能指定使用传输层协议的类型以及参数等
  • 网络通信发送数据时, ip地址指明目标主机, 端口号指明该主机上接受消息的特定进程.
